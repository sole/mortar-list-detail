List and details

What is this?

This is a template that presents a list of articles, and allows the user to expand the selected article and read its details by sliding the full view of the article from the right.

The list/detail paradigm is found in pretty much every mobile app.

How does it work?

When the page is loaded, we gather all <article> elements inside the #list element, and set up some listeners to detect when an article is touched.

Then when the user taps over an article, we copy its title and contents to the detail view, and slide to the right, in order to see the details.

You can have a look at app.js to see what is going on, code wise.

x-slidebox? x-slides? X-Tag!

We are using a few custom web components to build this template with clean markup.

Web components allow developers to define their own HTML elements that can be used as any other existing tag. For example, we are using the <x-slidebox> from X-Tags to lay out the two views (list and detail) and get the sliding behaviour, all with very few lines of code.

It can be installed (under certain platforms)

If you view this template with Firefox OS or a Firefox browser, you will see an Install button. This allows you to turn the template into a web app that can be installed in your device: you get the shortcut in your list of apps, and you can open and close it like any other app!

Open Web Apps require you to provide a file called manifest.webapp that specifies what the full name of the app is, a description, where the icon is, etc. It is called the manifest file, and we already wrote one for you.

Be sure to have a look at the manifest file and change as many values as required if you use this template as the base for your app!
